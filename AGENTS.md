# 全局工程指令 (Global Engineering Directives)

## 1. 核心原则
- **语言策略**:
  - **交互**: 默认使用中文。
  - **交付物**: 自动适配项目现有语言环境；英文仓库用英文。
- **风格**: 简洁、技术导向。直接给出结论、方案和验证结果。
- **先想后写**:
  - 动手前显式列出假设。不确定就问，不要沉默猜测。
  - 需求存在多种解读时，呈现选项，不得擅自选定。
  - 存在更简单方案时，主动提出。该 push back 就 push back。
- **决策边界**:
  - **自主决策**: Bug 修复、局部重构、单元测试补充、文档补充、低风险配置修正。
  - **必须请示**: 核心接口签名变更、数据库 Schema 修改、引入新依赖、跨模块大规模变动、潜在破坏性行为变更。
  - **冲突处理**: 仓库既有约定与本指令冲突时，以仓库约定为准。

## 2. 技能与工作流 (Skills & Workflow)
- **技能优先**:
  - 在开始开发任务前，必须尝试调用 `superpowers:using-superpowers`，
  - 若调用成功，遵循其工作流指引执行任务。
  - 若该技能不可用，**显式告知用户"superpowers 技能未检测到"**，再检查当前环境中是否存在可用的 workflow / skill / expert guidance。
  - 若存在相关能力，则优先调用并遵循其工作流。
  - 若不存在，则按同等工程标准手动执行：先澄清目标，再识别约束，再实施变更，最后完成验证。
- **目标驱动执行**:
  - 默认采用 TDD 或"先失败、后修复、再回归验证"推进。
  - 非测试类任务也应将"做什么"转化为"验证什么"：按 `[步骤] → verify: [检查方式]` 推进。
  - 文档、配置、迁移等不适合 TDD 的任务可豁免，但须说明采用的验证路径。

## 3. Git 规范
- 优先遵循仓库现有的提交规范和分支策略。无约定时使用以下默认规则：
- **Commit 格式**: `<type>(<scope>): <描述>`
- **语言**: 与项目主流语言保持一致；英文仓库使用英文，中文仓库使用中文。
- **Scope**: 受影响的目录名、模块名或子系统名。
- **Body**: 补充"为什么改"而非仅"改了什么"。
- **Breaking Change**: 涉及破坏性变更时显式标注 `BREAKING CHANGE:`。
- **分支**: `feature/<kebab-case>`，切换前确保工作区干净。

| type | 适用场景 |
| :--- | :--- |
| `feat` | 新功能 |
| `fix` | Bug 修复 |
| `test` | 测试相关 |
| `docs` | 文档、注释 |
| `refactor` | 重构，不改变外部行为 |
| `infra` | CI/CD、构建脚本、工程基础设施 |
| `dx` | 开发体验优化、工具脚本改进 |
| `chore` | 杂项维护、依赖更新 |

## 4. 简洁性原则
- 只写被要求的功能，不做投机性扩展。
- 单次使用的逻辑不要抽象成类/策略模式/配置化。
- 不添加未被要求的"灵活性"或"可配置性"。
- 如果 200 行能用 50 行解决，重写它。
- **自检**: "高级工程师会说这过度复杂吗？" 是 → 简化。

## 5. 代码质量底线
- **外科式修改**:
  - 遵循项目现有命名、结构、风格，即使你觉得有更好的写法。
  - 不得"顺手"改进相邻代码、注释或格式。不重构没问题的代码。
  - 发现不相关的死代码/技术债，**提出但不自行删除**。
  - 自己产生的孤儿（未使用的 import/变量/函数）必须清理。
  - **自检**: 每一行改动都能追溯到用户的需求吗？
- **安全性**:
  - 严禁硬编码 API Key、Token 或其他敏感信息。
  - 引入新依赖时评估必要性、维护状态与许可证风险。
- **完整性**:
  - 在支持显式类型标注且项目已有相关规范的语言中，补齐必要的类型信息。
  - 提交前主动运行项目已有的 lint / typecheck，修复自己引入的问题。
- **可维护性**:
  - 不为逻辑上不可达的路径写防御代码；但对可达路径不得静默吞掉异常。
  - 对大数据量操作需考虑算法复杂度，避免隐性 O(n²) 问题
  - 日志级别与项目既有约定保持一致，默认使用 error/warn/info/debug 分级。
  - 不保留临时性的 `print`、`console.log`、调试开关或注释掉的大段废弃代码。
