# 合并自检清单

只维护这一份清单。先做局部检查，再做全局检查；判词表的完整定义见 `SKILL.md`。

## 局部检查

- [ ] 每个段落是否只表达一个清楚的意思，首句是否承担本段或本节的结论；教程步骤是否明确操作目标。
- [ ] 使用的证据是否直接支撑该段结论，判断与推测是否写出依据和边界；证据不足与相反证据是否区分，认识状态和判断极性是否与 source bound 一致。
- [ ] 节间转场是否承接读者的下一个问题，而不是复述作者的执行顺序。
- [ ] 术语、数字口径和对象名称是否前后一致。
- [ ] 是否按“主体 + 判断/动作 + 关键条件/范围边界”扫描语义重复，而非只比较字面；同一完整命题是否只在一个权威位置展开；其他位置是否仅因承担摘要、过渡或证据等不同读者功能才保留，并压缩为引用或指向；本来很短、无法自然压缩的完整命题，是否避免因摘要和正文都想覆盖而换词完整复述；去重时是否保留必要证据。

## 阶段 B：独立风格校准

- [ ] 是否优先写成“主体 + 动作 + 结果”，而非用口号、营销语、比喻或抽象名词替代定义和证据。
- [ ] 是否按成稿阅读顺序逐个定位事实锁中的非通用缩写，并确认其首次出现处已按需解释或只保留全称；标题中的出现是否同样检查。
- [ ] 是否一段一意、一句一个主要判断或动作；必要的因果、条件和范围边界是否完整保留；是否把未验证、无数据、待审批或证据不足仅写为未知、待确认或证据边界，并仅在 source bound 明确给出冲突事实或相反证据时判定冲突；是否避免反向弱化已确认冲突。
- [ ] 仪式性开头和空泛结尾是否删除；是否避免把“更短”本身当成功标准。
- [ ] 材料内黑话是否仅在 source bound 明确支持时改成具体主体、动作与结果；证据不足时，是否保留带引号的源词，并对照事实锁按“主体、动作、对象、责任范围、强度”逐项说明未定义内容，或在不损失决策语义时删除纯空话；是否避免依据行业惯例增强职责判断。

## 阶段 C：对照事实锁复核

- [ ] **事实锁前提**：事实锁是否绑定原材料和已核实证据；每个 stable ID 是否只锁一个原子命题，将定义、动作、职责、因果、目标或阈值分别记录；字段是否包含 type、精确命题、限定信息、标准术语/标识符、source locator、direct/inference/unknown support mode，以及判断的依据与 status；黑话含义不明时，是否避免混入相邻 direct 事实，并以 unknown 和“未知”项按“主体、动作、对象、责任范围、强度”逐项记录。
- [ ] **正向事实槽**：是否逐 atomic stable ID 记录成稿位置，并确认单一命题、数值、条件、范围、否定/排他边界、术语/标识符、support mode、判断依据与 status 均保留。
- [ ] **反向主张槽**：是否按成稿顺序逐句拆出原子主张，并记录命题、direct/inference/unknown 与 source locator/依据；direct 的 locator 是否完整支持主体、关系、条件与强度，而非只列相关事实；inference 是否在成稿显式标注并保留依据与 status；unknown 是否没有写成结论。
- [ ] **可见顺序缩写槽**：是否逐个记录非通用缩写的首次可见位置及释义位置，标题同样计入；释义滞后是否记为 mismatch。
- [ ] **类型专项槽**：是否逐项对照每个保留黑话的主体、动作、对象、责任范围和强度未知维度；教程是否逐行核对状态转换表；重写是否逐项核对七条判词扫描结果；任一未知维度遗漏或教程未知动作未在正文披露，是否记为 mismatch。
- [ ] 是否逐项核对“源文黑话 → 风格稿表达”映射；更具体或更强的动词，以及主体、对象、责任强度或范围的变化，是否均记为 mismatch；判断依据中的证据关系、认识状态、极性或强度发生变化时，是否同样记为 mismatch。
- [ ] 四个适用复核槽是否全部填写、support mode 与来源一致且 mismatch 为 0；是否避免把只有 locator、没有语义支持核对的主张判为通过；同一问题句段一次局部修复后仍 mismatch 时，是否恢复 pre-style、事实正确的原文并停止继续改写该句段；任一槽未完成或失败时，是否避免在质检摘要中声称“全部覆盖”“未新增事实”或 `0 mismatch`。

## 全局检查

- [ ] 判词 1：第一屏是否给出中心意图；论证型有命题，任务型有任务承诺。
- [ ] 判词 2：是否按读者问题组织；只有教程操作步骤或因果证据才保留时间顺序。
- [ ] 判词 3：是否有明确主次与取舍，而不是平铺要点。
- [ ] 判词 4：关键选择是否解释为什么，并交代排除的主要备选。
- [ ] 判词 5：标题是否承载信息，而不是“背景/分析/总结”式空栏目名。
- [ ] 判词 6：信息密度和细节层级是否匹配目标读者。
- [ ] 判词 7：关键断言是否有出处或推断依据，推测是否被明确标注。
- [ ] 教程起草前是否逐个关键转换填写“前态 → 必要动作 → 后态 → 验证证据”；source bound 缺少必要动作时，动作栏是否记为未知并在正文单列待确认步骤或阻塞项；最终验证是否确认运行对象身份，而未把对象存在或后续成功信号当作已采用、已加载或已生效的替代证据。
- [ ] 重写模式是否在内部逐项记录判词 1 至 7 的命中位置或未命中，并在交付诊断中列出全部命中项及具体位置。
- [ ] 中心意图测试：论证型能否提炼一句可被反驳的命题；任务型能否提炼一个可验证的任务承诺。
- [ ] 标题连读测试：抽出全部标题连读，是否构成完整故事线或任务路径。
- [ ] 第一屏测试：只读第一屏，目标读者能否复述中心意图。
- [ ] 删减测试：逐节问“删掉这节，读者会损失什么”；答不出就删。

## 新读者测试

重要交付先预测 5-10 个读者问题，再创建不继承当前对话或任务历史的新上下文；接口支持时使用 no-history/no-fork。新上下文只传目标读者画像、问题清单和完整成稿，不传写作讨论、作者意图、自检结果或其他任务历史。问题应覆盖结论、依据、边界、风险和行动，例如：

1. 这份文档希望我接受什么判断或完成什么任务？
2. 这个决策的主要风险是什么，哪些事实支撑风险判断？
3. 结论在什么条件下不成立，下一步需要谁做什么？

执行时补齐读者画像和问题清单，将以下指令与完整成稿作为新上下文的全部输入：

```text
你是这份文档的目标读者：[读者一句话画像]。你只有这份文档，没有其他上下文。
通读后回答：1) 下列问题各用一两句回答：[问题清单]；
2) 哪些地方有歧义或可作两种解读；3) 文档假设了你已知道哪些它没交代的背景；
4) 有无内部矛盾。只基于文档作答，不要补充外部知识。
```

根据回答回到正文修订，直到关键问题能从文档中直接回答。没有 subagent 时，把指令和问题清单交给用户，在新会话手测；日常交付由作者以“读者只看得到成稿”为前提轻量自测。
