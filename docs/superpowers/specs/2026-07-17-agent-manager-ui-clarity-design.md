# Agent Manager 页面清晰度与一致性修订设计

**日期：** 2026-07-17
**状态：** 待用户复审
**范围：** 仅调整 Agent Manager Web 页面及对应测试；不修改 CLI、HTTP 数据结构、Skill/Instructions 写入语义或真实 HOME 状态。

## 1. 背景

当前页面功能完整，但真实环境中存在三类可用性问题：

- Skills 表格将完整描述直接铺在首列，行高不稳定，状态入口缺少统一尺寸。
- 个人约束列表没有列标题，路径、覆盖范围、状态和三个操作按钮缺少稳定对齐。
- 全局库存一次展示 166 条记录，却没有搜索、范围、工具、来源或异常筛选，也没有解释 `managed`、`plugin`、`built-in`、`local-copy` 的含义。

此外，本轮实现提交信息错误地使用了英文；历史整理阶段必须将实现提交的 subject 与 body 全部改为中文。

## 2. 目标与非目标

### 2.1 目标

- 让用户进入每个列表页后，能在 5 秒内理解“这里管理什么、当前是否正常、下一步能做什么”。
- Skills、个人约束和库存共享一致的表头、行高、状态标签和按钮体系。
- 库存默认展示“仓库受管 + 有异常标记”的记录，同时可筛选和查看全部记录。
- 保留当前详情抽屉、preview、confirm、apply 和安全边界。
- 所有新增中文界面文案使用用户能识别的业务含义，不直接暴露内部枚举作为主标签。

### 2.2 非目标

- 不增加新的后端接口或查询参数。
- 不改变库存扫描范围、重复检测或状态判定。
- 不增加分页、虚拟滚动、排序设置持久化或第三方前端依赖。
- 不重新设计总览拓扑的数据模型。

## 3. 视觉与布局原则

页面采用紧凑桌面管理工具风格：浅灰画布、白色内容面板、稳定分隔线和有限状态色。蓝色仅用于当前导航和主操作；绿色、橙色、红色仅表示状态。

- 内容区扩大到适合桌面表格的宽度，列表放在独立白色面板中。
- 表头固定视觉节奏；常规行保持统一最小高度，长文本截断为一行。
- 按钮统一为 32px 高度，并区分主按钮、次按钮和文字按钮。
- 路径使用等宽字体，默认单行省略，完整值放在 `title` 和详情抽屉中。
- 状态用“圆点 + 中文短标签”表达，不用整格按钮承担状态和操作两个职责。
- 唯一的识别性元素是“路由轨”：每个受管行在名称左侧显示细蓝线，表达其来源于当前仓库；异常行改用橙色线。其余装饰保持克制。

## 4. 页面结构

### 4.1 Skills

表格列固定为：

1. Skill：复选框、名称、slug；描述限制为一行。
2. Claude、Codex、Copilot、Antigravity：只显示状态标签和“查看”入口。

筛选行和批量操作行使用同一高度。批量操作始终右对齐，按钮文案保持“预览启用 / 预览停用 / 预览采用”。详情与单项操作继续在抽屉中完成。

### 4.2 个人约束

增加可见列标题：目标、加载位置、覆盖工具、状态、操作。

- 目标列显示 `shared`、`claude`、`codex` 等稳定 key。
- 路径列只显示紧凑路径，完整路径在详情中查看。
- 三个预览按钮固定宽度并靠右排列。
- Copilot Desktop 手动配置行使用相同列结构，不再作为跨列自由布局。

### 4.3 全局库存

库存顶部由三层组成：

1. 摘要：受管数量、异常数量、当前结果数、全部数量。
2. 范围切换：`受管 + 异常`（默认）、`全部`、`仅受管`、`仅异常`。
3. 筛选：搜索名称/slug/路径、工具、来源；提供“清除筛选”。

真实环境基线为：75 条受管、4 条带标记、166 条全部记录；“受管 + 异常”去重后为 77 条。页面必须从服务端返回的数据动态计算这些数字，不硬编码基线。

库存表格列为：Skill、来源、工具、状态、路径、操作。默认排序规则：异常优先，其次受管记录，再按名称、工具和路径排序。

来源枚举显示为：

| 服务端值 | 页面标签 | 说明 |
|---|---|---|
| `managed` | 仓库受管 | 由当前 lucas-skills 仓库管理 |
| `plugin` | 插件提供 | 来自工具已安装插件 |
| `built-in` | 工具内置 | 工具自身提供 |
| `local-copy` | 本地独立 | 本地存在但不由当前仓库管理 |

未知来源显示“其他来源”，并在详情抽屉保留原始值。异常标记必须显示中文标签；详情抽屉同时保留原始 flag，便于诊断。

## 5. 交互与状态

- 所有筛选在浏览器内作用于已加载库存，不新增网络请求。
- 刷新库存后保留当前筛选条件，并重新计算摘要和结果数。
- 空结果必须说明是“筛选后无结果”，并提供清除筛选按钮。
- 点击 Skill 名称或“查看”打开详情抽屉；点击表格空白区域不触发操作。
- 所有写操作仍必须先取得服务端 preview；页面不得根据表格状态自行推断写计划。
- 键盘焦点、忙碌态、dialog 焦点陷阱和 reduced-motion 约束保持现有行为。
