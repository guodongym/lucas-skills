# Agent Manager 控制台视觉重设计方案

- 日期：2026-07-18
- 状态：修订版待复审（v3：响应第二轮复审——摘要跳转契约改为锚定问题列表、测试口径改为 unittest 双门、功能色拆分文字 token）
- 基线：`feature/agent-manager-ui-polish@816b3c7`（信息架构修复 + 双通道状态反馈修复已完成的版本）
- 参考体系：腾讯 TDesign 设计语言（公开设计 token 与组件规范）

## 1. 现状诊断：为什么"不好看"

信息架构一轮修复后，页面已经"干净、正确、可用"，但审美停留在线框稿水平。具体病因：

1. **无色彩立场**。`#F4F6F8` 灰白底 + 白卡 + 1px 细灰线，"安静"执行成了"没有态度"。Route 蓝只出现在导航高亮和 2px 连线上，占画面比例不足 2%。
2. **无深度体系**。全页零投影，层级只靠细线围合，卡片、抽屉、确认框在视觉上是同一层，"生产级控制台"应有的空间感缺失。
3. **状态没有气场**。健康/警示状态用 7px 小点 + 彩色文字表达，在一屏 60 个状态格里形不成可扫读的色块节奏——这是"工程调试页"观感的最大来源。
4. **字体层级平**。Avenir Next 只用在两处标题，数字统计与正文字阶差距不足，大数字（15/15、75）没有仪表盘应有的分量。
5. **签名元素太轻**。拓扑是页面唯一的识别性元素，但 2px 灰线 + 白卡在灰底上，第一眼几乎不可见。

## 2. 参考体系调研结论

### 2.1 为什么选 TDesign 作主参照

- 定位吻合：TDesign 的核心场景就是中后台工具（TDesign Starter 即控制台模板），密度、组件形态、状态语义与本页面一致。
- 生产级完备：色彩有 1–10 全梯度、功能色带浅底/深字配套、投影分三级、字阶与间距成体系——正是现状缺的那几层。
- 气质匹配："理性、克制、可信"的企业级审美，符合本工具"安全执行变更"的心智。

### 2.2 引用方式（硬约束）

本项目 CSP 禁止外部资源、无构建链、不引入框架（既有 spec 非目标）。因此**不引入 tdesign 组件库**，只借鉴其设计 token 与组件视觉规范，全部手写原生 CSS 实现。token 值的官方梯度归属以 tdesign.tencent.com/design 为准，实施第一步须逐项校准；**对比度红线（文本 ≥4.5:1、非文本 ≥3.0:1）优先于梯度归属**——校准后若二者冲突，选满足红线的相邻档并记录偏差。

### 2.3 同类标杆的补充结论

本地开发者工具的优秀先例（Vite UI、Vitest UI、Drizzle Studio）共同点：一块高识别度的"主视觉区" + 其余界面高度克制。这印证方向：把大胆花在一处，其余交给 TDesign 的成熟秩序。

### 2.4 与既有 spec 的关系

本方案**替代** `2026-07-16-agent-manager-and-personal-instructions-design.md` §10.6 的全部视觉 token 规定（旧六色 token、"不使用投影"、"层级只靠空间/细线/字重"等条款自本方案批准起失效）。该文档 §10.7 的交互、响应式与可访问性约束**继续有效**，本方案 §6 是对其的增量补强而非冲突。实施与后续 review 以本方案为唯一视觉依据。

## 3. 设计方向与签名元素

**方向：TDesign 明亮生产级控制台 + 一块深色"路由面板"作为签名。**

整页采用 TDesign 明亮体系（专业、可信、久看不累）；唯独总览的拓扑区反转为深色面板——深海军蓝底上的发光路由线，像控制台中嵌入的一块真实网络设备面板。明暗反差制造记忆点，工程上只影响一个区块的样式，且深底恰好解决"连线太轻"的问题（同样 2px 的线，深底上对比度翻数倍）。

这是本方案唯一的审美冒险，其理由：拓扑承载页面的核心隐喻（仓库 → 路由 → Agent），值得也只有它值得被"点亮"。

| `#006C45` / `#E3F9E9`（健康徽章） | 5.88:1 |

基础功能色拆分文字 token 的原因：`#2BA471`、`#E37318`、`#D54941` 对白底实测仅 3.16:1、3.12:1、4.32:1，均不满足正文红线——只能用于填充、图标、路由线和大文本（摘要条 36px/600 数字适用 3.0:1 红线）；一切 ≤18px 正常字重的功能色文字一律使用 `--*-text` 深档。

### 4.2 字体与字阶

- 字族不变（系统栈 + SF Mono 路径），改**字阶与字重**：
- 字阶（TDesign 体系）：12 辅助 / 14 正文 / 16 小标题 / 20 区块标题 / 28 页标题 / **36 统计大数字**。
- 统计数字用 `font-variant-numeric: tabular-nums` + 600 字重——摘要条从"文字行"变成"仪表读数"。
- 页标题从当前 34px 巨字降为 28px，把空间还给内容。

### 4.3 空间、圆角、投影

- 间距 8px 栅格：卡片内边距 24，区块间 32，控件间 8/16。
- 圆角：卡片 9px、按钮与输入 6px、徽章 3px（TDesign 三档）。
- 投影（关键新增，替代"全靠细线"；TDesign 三档完整值，待官网校准）：
  - `--shadow-1`: `0 1px 10px rgba(0,0,0,.05), 0 4px 5px rgba(0,0,0,.08), 0 2px 4px -1px rgba(0,0,0,.12)` —— 卡片默认
  - `--shadow-2`: `0 3px 14px 2px rgba(0,0,0,.05), 0 8px 10px 1px rgba(0,0,0,.06), 0 5px 5px -3px rgba(0,0,0,.10)` —— hover 升起
  - `--shadow-3`: `0 6px 30px 5px rgba(0,0,0,.05), 0 16px 24px 2px rgba(0,0,0,.04), 0 8px 10px -5px rgba(0,0,0,.08)` —— 抽屉、确认框弹层
- 细线保留用于卡内分隔；卡与背景的边界交给投影。

## 5. 组件规范

### 5.1 状态徽章（改动最大、收益最大）

所有状态表达统一为 TDesign 浅底徽章形态：浅底色块 + 深档功能色文字 + 3px 圆角 + 12px/600。

- 已启用 → `--success-light` 底 + `--success-text` 字；未启用 → `--neutral-light` 底 + `--muted` 字；需要处理/冲突 → `--warning-light` 底 + `--warning-text` 字；错误 → `--error-light` 底 + `--error-text` 字。
- Skills 矩阵单元格 = 一枚徽章（hover 时整格升 `--shadow-1` 并加描边，提示可点开详情）。60 格从"彩色文字"变成可扫读的色块矩阵——健康时一片安静的浅绿，异常一眼定位。
- 现有"实心圆/方形"点保留在徽章内部作形状语义（色盲可辨）。

### 5.2 按钮体系（主次分级）

现状全部是灰描边按钮。改为三级：

- Primary（蓝实心）：每个上下文至多一个——"确认执行"、危险确认框的最终按钮（红实心）。
- Secondary（描边）：常规操作（启用/停用/转为直链）。
- Text（无边框）：低频辅助（清除筛选、设置步骤）。
- Danger（红描边，确认态红实心）："处理冲突"、"替换现有文件…"。描边与文字用 `--error-text`；确认态实心底也用 `--error-text` + 白字（6.81:1）——**不得**用 `--error` 做实心底（白字仅 4.32:1，低于红线）。

### 5.3 导航与顶栏

- 左栏加 16px 内联 SVG 线性图标（总览/Skills/约束/库存各一枚，语义组件生成，`aria-hidden`），高亮项蓝底浅色块（`--brand-light`）+ 左侧 3px 蓝条。
- 顶栏仓库路径截断显示（`~` 缩写 + title 全文），扫描时间与操作按钮右对齐一行，消除现在的换行挤压。

### 5.4 拓扑深色面板（签名区）

- 面板：`--panel-deep` 底、12px 圆角、`--shadow-1`，内边距 32。
- 深底配套 token（对比度为脚本实测；非文本红线 3.0:1、文本红线 4.5:1）：

| token | 值 | 用途 | 实测（vs 面板底） |
| --- | --- | --- | ---: |
| `--panel-ink` | `#C8D6EE` | 面板正文、节点名 | 11.26:1 |
| `--panel-muted` | `#8DA3C4`（自定） | 面板辅助文字、路径 | 6.43:1 |
| `--panel-card` | 底 `rgba(255,255,255,.06)` + 1px `rgba(255,255,255,.14)` 描边 | 工具卡，悬停提亮 | — |
| `--panel-line-trunk` | `#618DFF`（Blue 5） | 干线 | 5.32:1 |
| `--panel-line-ok` | `#56C08D`（Green 4） | 健康路由线 | 7.33:1 |
| `--panel-line-warn` | `#FA9550`（Orange 4） | 异常路由线 | 7.46:1 |
| `--focus-ring-dark` | `#8EABFF`（Blue 4） | 面板内键盘焦点 | 7.41:1 |

- 深底徽章（三态；背景 = 亮档色 16% 透明度叠加面板底，附 1px `rgba(亮档色,.4)` 描边）：

| 状态 | 文字 | 背景 | 合成后实际底色 | 实测 |
| --- | --- | --- | --- | ---: |
| 健康 | `#56C08D` | `rgba(86,192,141,.16)` | `#173947` | 5.44:1 |
| 警示 | `#FA9550` | `rgba(250,149,80,.16)` | `#31323E` | 5.72:1 |
| 错误 | `#FF9285` | `rgba(255,146,133,.16)` | `#323146` | 5.83:1 |

- 焦点规则：面板内可聚焦元素（"处理"按钮等）改用 `outline: 2px solid var(--focus-ring-dark); outline-offset: 2px`，**不沿用**全局半透明品牌蓝焦点环——该环对 `#0B1F3A` 仅 1.27:1，深底上不可见。
- 路由线：2px，Skills 实线 / 约束虚线不变。深底上自然呈现"发光"感，无需 glow 特效（`prefers-reduced-motion` 均安全）。
- 小屏纵向堆叠时面板保持深色，连线隐藏规则沿用现状。

### 5.5 其余组件

- 摘要条：四格改为"36px tabular 数字 + 12px 标签"的仪表形态，冲突/问题非零时数字用 `--warning`（36px/600 属大文本，3.0:1 红线下达标；可点击行为见 §6）。
- 表格：表头 12px/600/`--ink-secondary`、行 hover `--canvas` 底；库存来源徽章沿用新徽章体系。
- 抽屉与确认框：`--shadow-3` + 9px 圆角；确认框标题 20px；危险确认框顶部 4px 红条。
- 空状态：图标（内联 SVG）+ 主句 + 行动句，替代现在的单行灰字。

## 6. 交互闭环补强（唯一的交互层改动）

基线 816b3c7 已具备"问题 → 视图"的跳转骨架：拓扑异常通道的"处理"按钮和"需要处理"列表的"查看"按钮都会切换视图并把焦点移到主内容。评估结论：**流程不需要重构**（扫描 → 选择 → 确认 → 应用、指纹事务、抽屉详情均经过一轮修复与 review），缺的只是跳转**落点精度**——切过去之后用户仍要在 60 个状态格或约束列表里自己找目标。只做三项定向补强：

1. **问题记录携带落点**："需要处理"列表与拓扑"处理"按钮的跳转记录携带 `{view, slug/key}`：跳到 Skills 视图时状态筛选器自动置为"需要处理"、搜索框置为该 slug；跳到个人约束视图时滚动定位到对应条目。没有对应目标行的记录（仓库/约束扫描问题、未完成事务）不带"查看"按钮，在列表内直接展示路径与消息（沿用现状）。
2. **摘要数字锚到问题列表**：摘要条"冲突 / 扫描问题"非零时数字格变为按钮，点击滚动定位并聚焦"需要处理"列表。不做直接跳转到目标行——`conflicts` 是 Skills 与个人约束跨通道相加，`issues` 还含仓库/约束扫描问题、损坏目标与未完成事务，聚合数字没有唯一目标行。
3. **落点高亮**：经第 1 项跳转后目标行短暂高亮后淡出（沿用 180ms 过渡；`prefers-reduced-motion` 下静态高亮、不做淡出动画），焦点行为沿用现状。

评估过、明确不做：hash 路由与刷新后状态持久化（本地单页工具，页面级刷新场景少）；批量操作栏常显（选中即现是标准上下文操作模式）；抽屉改独立页（既有 spec 决策）。除以上三项外，信息架构、操作流程、文案与后端契约零改动。

## 7. 非目标

- 不引入 tdesign-vue/react 组件库、图标字体、任何外部资源或构建步骤（沿用既有 spec 约束）。
- 不做暗色模式全局主题（深色仅限拓扑面板；全局暗色留待后续独立方案）。
- 除 §6 三项闭环补强外，不改信息架构、交互流程、文案与后端契约。
- 不新增动画：仅保留现有 180ms 过渡，投影 hover 与落点高亮淡出使用同一时长。

## 9. 验收

1. 实施前对照 tdesign.tencent.com/design 校准 §4.1/§4.3 全部 token 归属；任何色值变更后重跑对比度脚本，红线（文本 4.5:1、非文本 3.0:1）优先于梯度归属，偏差记录进提交说明。
2. 全部功能色组合按 §4.1 与 §5.4 表逐项实测达标（表中数值为基准）；保留形状语义。
3. 双测试门全绿（以实施时实际数为准）：`uv run python -m unittest tests.test_agent_manager_web -q`（Web 专项，当前 50 项）；`uv run python -m unittest discover -s tests -p 'test_*.py' -q`（全量，当前 251 项）。另 `node --check`；无外部请求；CSP 不放宽。
4. 真实 HOME 只读浏览：1440×900 与 390×844 两档截图对比本方案逐节核对；console 零错误。
5. 健康状态页与异常状态页各截一屏，确认"一眼判断健康度"在新视觉下成立（异常色块可 3 秒内定位）。
6. 键盘焦点回归：浅色区沿用全局焦点环；深色面板内焦点环达 §5.4 实测值；`prefers-reduced-motion` 不弱化。
7. 闭环验收：非零摘要数字 → "需要处理"列表 ≤1 次点击；列表中可定位记录 → 已筛选/已定位的目标行再 ≤1 次点击，且有落点高亮；无目标行的记录在列表内完整展示路径与消息。
